## MHA with KV Cache

- past_k, past_v: [B, num_head, past_len, head_dim]
- 流程：
  - proj -> split heads
  - rope
  - add kv
  - Q@K -> mask -> softmax -> @v -> merge head -> proj return

```python
class MHA_KV(nn.Module):
  def __init__(self, num_head, dim):
    super().__init__()
    self.num_head = num_head
    self.dim = dim
    self.head_dim = dim // num_head
    self.q_proj = nn.Linear(dim, dim, bias=False)
    self.k_proj = nn.Linear(dim, dim, bias=False)
    self.v_proj = nn.Linear(dim, dim, bias=False)
    self.out_proj = nn.Linear(dim, dim, bias=False)
    self.rope = RotaryEmbedding(self.head_dim)

  def forward(self, x, past_kv):
    B, T, _ = x.shape
    q = self.q_proj(x).view(B, T, self.num_head, -1).transpose(1, 2)
    k = self.k_proj(x).view(B, T, self.num_head, -1).transpose(1, 2)
    v = self.v_proj(x).view(B, T, self.num_head, -1).transpose(1, 2)
    # rope
    past_kv_len = past_kv[0].shape[2] if past_kv is not None else 0
    q = self.rope.rotate_queries_or_keys(q, offset=past_kv_len)
    k = self.rope.rotate_queries_or_keys(k, offset=past_kv_len)
    # add previous kv cache
    if past_kv is not None:
      past_k, past_v = past_kv
      k = torch.cat([past_k, k], dim=2)
      v = torch.cat([past_v, v], dim=2)
    # q@k/sqrt(d)
    attn = q @ k.transpose(-1, -2) / (self.head_dim ** 0.5)
    # mask
    if T > 1:
      mask = torch.ones(T, k.shape[2], device=x.device).tril(diagonal=past_kv_len).bool()
      attn = attn.masked_fill(~mask, float('-inf'))
    # softmax
    attn = attn.softmax(-1)
    # @v
    out = (attn @ v).transpose(1, 2).contiguous().view(B, T, -1)

    return self.out_proj(out), (k, v)
```



## Next Token generate

- Input: [model, prompts, max_token_len, eos_id, past_kv]
  - prompts [B, n]
- prefill: (past_kv, prompts) -> model -> logits, past_kv
  - Next id (sample) -> out[] (check eos)
- Decode: (past_kv, new_token)
  - Next id (sample) -> out[] (check eos)

```python
@torch.no_grad()
def generate(model, input_ids, max_new_tokens, past=None, eos_id=None, **kw):
  B = input_ids.shape[0]                           # input_ids: [B, n]
  # prefill: 整段 prompt 一次前向, 写进 KV cache
  logits, past = model(input_ids, past)            # logits [B, n, V]; past 可空(新会话)/非空(多轮复用)
  next_id = sample(logits[:, -1], **kw)            # logits[:,-1] [B, V] → next_id [B, 1]
  out = [next_id]
  finished = torch.zeros(B, dtype=torch.bool, device=input_ids.device)   # 各序列是否已完成
  if eos_id is not None:
    finished = next_id.squeeze(1) == eos_id        # prefill 第一个 token 也可能是 EOS
  # decode: 每步只输入上一步产出的 1 个 token
  for _ in range(max_new_tokens - 1):
    logits, past = model(next_id, past)            # 复用历史 KV, 只算新 token
    next_id = sample(logits[:, -1], **kw)          # [B, 1]
    if eos_id is not None:
      next_id = next_id.masked_fill(finished[:, None], eos_id)  # 已停的填 EOS, 不再附加新 token
      finished |= (next_id.squeeze(1) == eos_id)                # 累积: 一旦停就永远算停 (异步结束)
    out.append(next_id)
    if eos_id is not None and finished.all():      # 全停才退
      break
  return torch.cat([input_ids, *out], dim=1), past # [B, n + 生成长度], past 给下一轮复用
```

Sampling pipeline：temperature → top-k → top-p → 按概率抽 1 个

```python
@torch.no_grad()
def sample(logits, temperature=1.0, top_k=None, top_p=None):   # logits: [B, V]
  if temperature == 0:                             # greedy (可复现)
    return logits.argmax(-1, keepdim=True)         # [B, 1]
  logits = logits / temperature
  if top_k:                                        # 只留最高的 k 个
    kth = logits.topk(top_k, dim=-1).values[..., -1, None]    # 第 k 大的门槛 [B, 1]
    logits = logits.masked_fill(logits < kth, float('-inf'))
  if top_p:                                        # 留累计概率到 p 的最小集合
    s, si = logits.sort(descending=True, dim=-1)   # s/si: [B, V] 排序后的值 / 原下标
    cum = s.softmax(-1).cumsum(-1)                 # [B, V] 累计概率
    drop = cum > top_p
    drop[..., 1:] = drop[..., :-1].clone()         # 右移一位, 保留刚越过 p 的那个
    drop[..., 0] = False
    s = s.masked_fill(drop, float('-inf'))
    logits = torch.full_like(logits, float('-inf')).scatter(-1, si, s)   # 散回原词表序 [B, V]
  return torch.multinomial(logits.softmax(-1), 1)  # [B, V] 概率 → 抽 1 个 → [B, 1]
```



## Continuous Batch

> 由于变长最好使用 kernel 计算，所以这里 P / D 中的 1 step 并没有实现真正意义上的并行

- Input：model, prompts, max_batch, eos_id, max_len
- Idea：3 个 queue — waiting, active, done
  - waiting init：存 prompt, kv, out
  - **while waiting or active**：
    - 拉新请求 prefill
    - decode 1 step
    - 完成的踢出

```python
@torch.no_grad()
def serve(model, prompts, max_batch, eos_id, max_len, **kw):
  waiting = [{"ids": p, "kv": None, "out": []} for p in prompts]
  active, done = [], []

  while waiting or active:
    # add new prefill
    while waiting and len(active) < max_batch:
      s = waiting.pop(0)
      logits, s["kv"] = model(s["ids"], s["kv"])
      s["out"].append(sample(logits[:, -1], **kw))
      active.append(s)
    # decode 1 step
    for s in active:
      logits, s["kv"] = model(s["out"][-1], s["kv"])
      s["out"].append(sample(logits[:, -1], **kw))
    # move done (diverge)
    keep = []
    for s in active:
      (done if s["out"][-1].item() == eos_id or len(s["out"]) >= max_len else keep).append(s)
    active = keep

  return done
```



## Chunked Prefill

> 在 continuous batching 上加一手：长 prefill 切定长 chunk，每 iter 把"一块 chunk + 在跑的 decode"拼进**同一次 forward**。
>
> 省的是 **TPOT**（长 prefill 不再独占一 iter 卡住 decode）+ decode 白嫖 prefill 的权重读（memory-bound → compute-bound）。

- 相对 serve 改两点：① 每条多带 prefill 游标 `cur`；② 每 iter 在 `token_budget` 内：prefilling 切 chunk、decoding 各捎带 1 token，一起 forward
- 核心就是那段 budget 循环（其余 while / admit / evict 同 serve）

```python
# ★ = 相对 continuous batch 的 serve 新增/改动
@torch.no_grad()
def serve_chunked(model, prompts, eos_id, token_budget=2048, max_batch=64, max_new=256, **kw):  # ★ token_budget
  waiting = [{"ids": p, "cur": 0, "kv": None, "out": []} for p in prompts]   # ★ 多带 prefill 游标 cur
  running, done = [], []
  while waiting or running:
    while waiting and len(running) < max_batch:
      running.append(waiting.pop(0))                 # ★ admit 不再立刻整段 prefill, 只入队 (cur=0)
    # ★★ 以下 budget 循环整段是新增: 替代 serve 的"admit 整段 prefill + 单独 decode 循环"
    budget = token_budget                            # ★ prefill chunk 与 decode 共抢
    for s in running:
      if s["cur"] < len(s["ids"]):                   # ★ prefilling → 切一块 chunk (多 token)
        n = min(len(s["ids"]) - s["cur"], budget)
        if n == 0: continue
        toks = s["ids"][s["cur"]:s["cur"]+n]; s["cur"] += n; budget -= n   # ★ 推进游标、扣预算
      elif budget:                                   # ★ decoding → 捎带 1 token
        toks = s["out"][-1]; budget -= 1
      else:
        continue
      logits, s["kv"] = model(toks, s["kv"])         # 真实: 一次 varlen forward; 逐条仅表义
      if s["cur"] >= len(s["ids"]):                  # ★ 只有 prefill 完 / decode 步才出 token
        s["out"].append(sample(logits[:, -1], **kw))
    keep = []                                        # evict 同 serve, 仅完成判断用 cur (已 prefill 完)
    for s in running:
      (done if s["cur"] >= len(s["ids"]) and (s["out"][-1].item() == eos_id or len(s["out"]) >= max_new) else keep).append(s)
    running = keep
  return done
```

追问要点：FFN / Linear 层 P+D 拼一个 GEMM（省时间主力，权重读摊薄）｜ MHA 层 P / D pattern 不同、可能分 kernel（P: FA+KV / D: flash-decoding）｜ token_budget 大 → TTFT 降 TPOT 升 ｜不写完整、写 budget 循环即可。



## KV Cache 类（连续 buffer 版）

> 上面 MHA_KV 用 torch.cat 拼 KV（简单）；进阶是**预分配定长 buffer + 游标写入**（真实框架做法，避免每步重分配）。被问"怎么管 KV cache"时写这个。下面 PagedAttention 再把它升级成 block 存储。

- 关键点：① **缓存的是投影后的 k / v**（不是原始 x）；② 定长 buffer 一次分配、游标 `len` 写入，支持 prefill (T>1) 和 decode (T=1)；③ 满了 **assert 报错**，别静默丢
- 用法：把 MHA_KV 里的 `torch.cat([past_k, k])` 换成 `self.cache.update(k, v)`（k / v 是投影 + 分头后的）

```python
class KVCache:
  def __init__(self, max_len, num_heads, head_dim):
    self.max_len, self.nh, self.hd = max_len, num_heads, head_dim
    self.k = self.v = None
    self.len = 0
  def update(self, k, v):                       # k, v: [B, nh, T, hd] (T=n prefill / 1 decode)
    B, _, T, _ = k.shape
    if self.k is None:                          # 懒分配定长 buffer
      self.k = torch.zeros(B, self.nh, self.max_len, self.hd, dtype=k.dtype, device=k.device)
      self.v = torch.zeros_like(self.k)
    assert self.len + T <= self.max_len, "KV cache overflow"   # 别静默丢
    self.k[:, :, self.len:self.len+T, :] = k    # 游标写一段 (prefill T>1 也行)
    self.v[:, :, self.len:self.len+T, :] = v
    self.len += T
    return self.k[:, :, :self.len, :], self.v[:, :, :self.len, :]   # 返回已写入部分
```

> 易错 / 追问：① 别用 `nn.MultiheadAttention`（内部会对整段重投影 → KV cache 白做），要手写 attention 缓存投影后的 k / v；② 这是**每序列一块连续 buffer**，一次预留 `max_len` → 有预留浪费 / 内部碎片，**正是 PagedAttention 要优化掉的**。



## PagedAttention

> 以概念为主、代码兜底（系统题，基本不手写 kernel）

- **一句话**：KV cache 版的 OS 虚拟内存——KV 切定长块，块表把逻辑块映射到**非连续**物理块、按需分配，消除碎片。
- **只对 KV 分页**：权重固定连续、激活临时，只有 KV 又大又变长又不齐 → 碎片全来自它。
- **OS 类比**（最佳答法）：

| OS 虚拟内存 | PagedAttention |
| --- | --- |
| 虚拟页 page | 逻辑 KV 块 |
| 物理帧 frame | 物理 KV 块（HBM） |
| 页表 page table | 块表 block table |
| 共享页 / COW | 块共享 + 引用计数 |
| 换页到磁盘 | swap 到 CPU |

- **三类碎片**：内部（最后一块没填满）/ 预留（按 max 预留、现在空着）/ 外部（变长分配留空洞）→ 分页把外部消掉、预留归零、内部限制在 <1 块。
- **共享**：块表指同一物理块即跨序列共享（并行采样 / beam / prefix cache），写时 **COW**（refcount>1 先复制再写）。
- **kernel 要重写**：KV 不连续 → kernel 多接收 `block_table`，循环单位 = 块、整块进 SRAM + online softmax 跨块累加（= paged FA）；decode query=1，主循环只在 KV 块维。

**block 版：PagedKVCache（在 KVCache 上加分页）**

> 把 KVCache 的"每序列一块连续 buffer"换成"**所有序列共享一个块池 + 每序列一张块表**"：要 KV 就按需领块、块在物理上不连续、靠块表寻址。
>
> 省的是**预留浪费**：不再一次性预留 max_len，用多少领多少；还顺带能跨序列共享块（prefix cache）。

- Input：`num_blocks, block_size, num_heads, head_dim`
- 写法（update：写一条序列的新 token，对照 KVCache 的改动）：
  - 取这条序列的**块表 blocks** 和**已填数 filled**（KVCache 是单一 buffer + 游标，这里是每序列一张表）
  - 逐 token：到块边界（`filled % block_size == 0`）就从 `free_blocks` 领一块（KVCache 是直接写游标，这里要先有块）
  - 定位 **(块号, 块内偏移)** 写入 → `filled += 1`
  - 按块表把散块 **gather** 成完整 [nh, L, hd] 返回（真实在 attention kernel 里按块表 gather）

```python
class PagedKVCache:
  def __init__(self, num_blocks, block_size, num_heads, head_dim):
    self.block_size = block_size
    self.k_pool = torch.zeros(num_blocks, num_heads, block_size, head_dim)  # 共享块池
    self.v_pool = torch.zeros_like(self.k_pool)
    self.free_blocks = list(range(num_blocks))   # 空闲块 id
    self.block_table = {}                        # seq_id -> [块 id, ...]
    self.filled = {}                             # seq_id -> 已填 token 数

  def update(self, seq_id, k, v):                # k, v: [nh, T, hd]  这条序列的新 token
    blocks = self.block_table.setdefault(seq_id, [])
    filled = self.filled.get(seq_id, 0)
    for t in range(k.shape[1]):                  # 逐个新 token 写入
      if filled % self.block_size == 0:          # 到块边界 → 领一块新块
        blocks.append(self.free_blocks.pop())    # 没块就 OOM → 抢占 / swap
      blk = blocks[-1]                           # 当前块
      offset = filled % self.block_size          # 块内第几个槽
      self.k_pool[blk, :, offset] = k[:, t]
      self.v_pool[blk, :, offset] = v[:, t]
      filled += 1
    self.filled[seq_id] = filled
    # 按块表 gather 出完整 K/V（真实在 attention kernel 里做，这里 cat 仅表义）
    k_full = torch.cat([self.k_pool[b] for b in blocks], dim=1)[:, :filled]   # [nh, filled, hd]
    v_full = torch.cat([self.v_pool[b] for b in blocks], dim=1)[:, :filled]
    return k_full, v_full
```

> 再加两样就是完整 manager（基本不会让完整写）：① **refcount + share()**（块表指同一物理块 + 引用计数 → 跨序列共享 / prefix cache，写时 COW）；② **swap / 抢占**（free_blocks 空时把某序列的块换出到 CPU）。



## Prefix Cache

- `get_blocks_hash`
  - input_ids, block_size → list: blocks_hash
  - call: `hash(prev_hash, block_tuple)`
- `get_prefix_cache`
  - input_ids, block_size, cache → list: prefix_cache

```python
def get_blocks_hash(input_ids, block_size):
  blocks_hash, prev_hash = [], None
  for i in range(len(input_ids) // block_size):
    # get this block
    block = tuple(input_ids[i*block_size: (i+1)*block_size])
    prev_hash = hash((prev_hash, block))
    blocks_hash.append(prev_hash)
  return blocks_hash

def get_prefix_cache(input_ids, block_size, cached):
  # get hashs
  blocks_hash = get_blocks_hash(input_ids, block_size)
  prefix_cache = []
  # match prefix
  for h in blocks_hash:
    if h not in cached:
      break
    prefix_cache.append(cached[h])
  return prefix_cache
```
